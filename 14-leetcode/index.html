⚡ LeetCode Speed Prep

Los 20 problemas Easy más frecuentes resueltos con patrones memorizables. Meta: completarlos todos en menos de 40 minutos.

20 Problemas 10 Patrones Plan 7 Días Meta: Sub-40 min
Regla operativa: En este challenge importa el throughput. Si la idea no aparece en 20–30 segundos, activá el patrón memorizado. Si el código ya pasa, no lo embellezcas. Si falla un test, corregí el bug puntual y seguí. Usá nombres cortos y escritura continua.
Los 10 Patrones Base

Estos patrones cubren el 100% de los 20 problemas. Memorizalos hasta poder reescribirlos sin mirar.

🔁

1 · Doble For Directo

Cuándo: array chico, solución obvia y segura, prioridad en entender rápido.
Idea: agarro un número, lo comparo con todos los que están después.

for (let i = 0; i < nums.length; i++) {
  for (let j = i + 1; j < nums.length; j++) {
    if (nums[i] + nums[j] === target) return [i, j];
  }
}
🗺️

2 · Map / Complemento

Cuándo: necesitás saber si ya viste un valor, "buscar complemento".
Idea: guardo lo que vi; para el actual pregunto si lo que falta ya apareció.

const vistos = new Map();
for (let i = 0; i < nums.length; i++) {
  const falta = target - nums[i];
  if (vistos.has(falta)) return [vistos.get(falta), i];
  vistos.set(nums[i], i);
}
📚

3 · Stack (Pila)

Cuándo: paréntesis, abrir/cerrar, LIFO.
Idea: si abro, guardo; si cierro, tiene que coincidir con lo último abierto.

const pares = { ")": "(", "]": "[", "}": "{" };
const pila = [];
for (const c of s) {
  if ("([{".includes(c)) pila.push(c);
  else if (pila.pop() !== pares[c]) return false;
}
return pila.length === 0;
🎯

4 · Binary Search

Cuándo: array ordenado, encontrar posición, partir por la mitad.
Idea: miro el medio; si es chico me voy a la derecha, si alcanza me quedo.

let l = 0, r = nums.length;
while (l < r) {
  const m = Math.floor((l + r) / 2);
  if (nums[m] < target) l = m + 1;
  else r = m;
}
return l;
🔤

5 · Linear String Scan

Cuándo: sumar valores, comparar con el siguiente, contar letras.
Idea: voy letra por letra, decido en cada paso, acumulo la respuesta.

let resultado = 0;
for (let i = 0; i < s.length; i++) {
  // comparar s[i] con s[i+1] si necesitás lookahead
  resultado += valorDe(s[i]);
}
return resultado;
👇👇

6 · Two Pointers

Cuándo: arrays ordenados, compactar, borrar sin delete, escribir al frente.
Idea: un puntero recorre, el otro marca dónde escribir lo válido.

let write = 1;
for (let i = 1; i < nums.length; i++) {
  if (nums[i] !== nums[i - 1]) {
    nums[write] = nums[i];
    write++;
  }
}
return write;
🔗

7 · Merge de Listas (Dummy Node)

Cuándo: dos listas ordenadas que hay que unir.
Idea: dummy node evita casos especiales al principio; siempre avanzo el menor.

const dummy = new ListNode(0);
let cur = dummy;
while (l1 && l2) {
  if (l1.val < l2.val) { cur.next = l1; l1 = l1.next; }
  else { cur.next = l2; l2 = l2.next; }
  cur = cur.next;
}
cur.next = l1 || l2;
return dummy.next;
🌳

8 · DFS en Árboles

Cuándo: recorrer árboles, inorder/preorder/postorder.
Idea: función recursiva; si no hay nodo termina; procesa en el orden correcto.

const resultado = [];
const dfs = (nodo) => {
  if (!nodo) return;
  dfs(nodo.left);         // inorder
  resultado.push(nodo.val);
  dfs(nodo.right);
};
dfs(root);
return resultado;
🪞

9 · Comparar Dos Árboles

Cuándo: same tree, mirror tree, verificar estructura y valores.
Idea: ambos null = true; uno null = false; valores distintos = false; recursión.

function mismoArbol(a, b) {
  if (!a && !b) return true;
  if (!a || !b) return false;
  if (a.val !== b.val) return false;
  return mismoArbol(a.left, b.left)
      && mismoArbol(a.right, b.right);
}
📈

10 · Fibonacci / DP Mínima

Cuándo: cada respuesta depende de las dos anteriores (Climbing Stairs).
Idea: no necesitás guardar todo, solo las últimas dos respuestas.

if (n <= 2) return n;
let anterior = 1, actual = 2;
for (let i = 3; i <= n; i++) {
  const sig = anterior + actual;
  anterior = actual;
  actual = sig;
}
return actual;
Los 20 Ejercicios

Organizados por categoría. Cada ficha tiene la idea mínima, el patrón, el pitfall principal y el tiempo objetivo.

📦 Arrays & Strings

Easy 1 · Two Sum

Patrón: Doble for directo
Idea: recorrés con dos for y comparás pares.
Pitfall: arrancar j en 0 en vez de i+1 hace trabajo repetido.
Objetivo: ≤ 90s

var twoSum = function(nums, target) {
  for (let i = 0; i < nums.length; i++) {
    for (let j = i + 1; j < nums.length; j++) {
      if (nums[i] + nums[j] === target) return [i, j];
    }
  }
};

"Pruebo cada número con los que tiene después. Si alguno suma el target, devuelvo sus posiciones."

Easy 26 · Remove Duplicates from Sorted Array

Patrón: Two pointers
Idea: puntero lento marca la zona única, rápido explora.
Pitfall: arrancar mal el índice y pisar la primera posición.
Objetivo: ≤ 90s

var removeDuplicates = function(nums) {
  let write = 1;
  for (let i = 1; i < nums.length; i++) {
    if (nums[i] !== nums[i - 1]) {
      nums[write] = nums[i];
      write++;
    }
  }
  return write;
};

"Siempre que encuentro algo distinto al anterior, lo escribo en la posición del write."

Easy 27 · Remove Element

Patrón: Two pointers / write index
Idea: compactás los distintos de val al frente.
Pitfall: querer borrar elementos en vez de reescribir.
Objetivo: ≤ 90s

var removeElement = function(nums, val) {
  let write = 0;
  for (let i = 0; i < nums.length; i++) {
    if (nums[i] !== val) {
      nums[write] = nums[i];
      write++;
    }
  }
  return write;
};

Easy 35 · Search Insert Position

Patrón: Binary search
Idea: binary search clásico con respuesta en l.
Pitfall: off-by-one en el while o devolver r.
Objetivo: ≤ 90s

var searchInsert = function(nums, target) {
  let l = 0, r = nums.length;
  while (l < r) {
    const m = Math.floor((l + r) / 2);
    if (nums[m] < target) l = m + 1;
    else r = m;
  }
  return l;
};

Easy 58 · Length of Last Word

Patrón: String cleanup
Idea: recortás espacios y tomás el último split.
Pitfall: no usar trim() y contar espacios finales.
Objetivo: ≤ 90s

var lengthOfLastWord = function(s) {
  const words = s.trim().split(" ");
  return words[words.length - 1].length;
};

Easy 66 · Plus One

Patrón: Carry simulation
Idea: propagás carry desde el final; si todo era 9, agregás 1.
Pitfall: olvidarte del caso [9,9,9].
Objetivo: ≤ 90s

var plusOne = function(digits) {
  for (let i = digits.length - 1; i >= 0; i--) {
    if (digits[i] < 9) { digits[i]++; return digits; }
    digits[i] = 0;
  }
  return [1, ...digits];
};

Easy 88 · Merge Sorted Array

Patrón: Two pointers desde el final
Idea: merge desde el final para no pisar nums1.
Pitfall: hacerlo desde adelante y destruir datos útiles.
Objetivo: ≤ 120s

var merge = function(nums1, m, nums2, n) {
  let p1 = m - 1, p2 = n - 1, pos = m + n - 1;
  while (p2 >= 0) {
    if (p1 >= 0 && nums1[p1] > nums2[p2]) {
      nums1[pos--] = nums1[p1--];
    } else {
      nums1[pos--] = nums2[p2--];
    }
  }
};

"Lleno nums1 desde el final. Así nunca piso valores útiles."

🔤 Strings Específicos

Easy 13 · Roman to Integer

Patrón: Linear string scan
Idea: sumás o restás según el valor siguiente.
Pitfall: olvidar el caso sustractivo (IV, IX, XL...).
Objetivo: ≤ 90s

var romanToInt = function(s) {
  const val = {I:1,V:5,X:10,L:50,C:100,D:500,M:1000};
  let resultado = 0;
  for (let i = 0; i < s.length; i++) {
    if (val[s[i]] < val[s[i+1]]) resultado -= val[s[i]];
    else resultado += val[s[i]];
  }
  return resultado;
};

Easy 14 · Longest Common Prefix

Patrón: Shrinking prefix
Idea: usás la primera palabra como base y recortás mientras no matchee.
Pitfall: llamar indexOf con el orden incorrecto o no cubrir array vacío.
Objetivo: ≤ 90s

var longestCommonPrefix = function(strs) {
  let prefix = strs[0];
  for (let i = 1; i < strs.length; i++) {
    while (strs[i].indexOf(prefix) !== 0) {
      prefix = prefix.slice(0, -1);
      if (!prefix) return "";
    }
  }
  return prefix;
};

Easy 28 · Find Index of First Occurrence

Patrón: Built-in string lookup
Idea: usás indexOf directamente.
Pitfall: reinventar substring search y perder tiempo.
Objetivo: ≤ 90s

var strStr = function(haystack, needle) {
  return haystack.indexOf(needle);
};

Easy 67 · Add Binary

Patrón: Two pointers + carry
Idea: suma de derecha a izquierda con carry.
Pitfall: no convertir chars a números o olvidar carry final.
Objetivo: ≤ 120s

var addBinary = function(a, b) {
  let i = a.length-1, j = b.length-1, carry = 0, res = "";
  while (i >= 0 || j >= 0 || carry) {
    const suma = (i>=0 ? +a[i--] : 0) + (j>=0 ? +b[j--] : 0) + carry;
    carry = Math.floor(suma / 2);
    res = (suma % 2) + res;
  }
  return res;
};

🔢 Math

Easy 9 · Palindrome Number

Patrón: Math / reverse integer
Idea: comparás el número con su versión invertida.
Pitfall: negativos nunca son palíndromos; no olvidarte de Math.floor.
Objetivo: ≤ 90s

var isPalindrome = function(x) {
  if (x < 0) return false;
  let original = x, invertido = 0;
  while (x > 0) {
    invertido = invertido * 10 + x % 10;
    x = Math.floor(x / 10);
  }
  return original === invertido;
};

Easy 69 · Sqrt(x)

Patrón: Binary search
Idea: buscás por binary search el mayor m con m*m ≤ x.
Pitfall: overflow mental o condición mal cerrada.
Objetivo: ≤ 120s

var mySqrt = function(x) {
  let l = 0, r = x;
  while (l < r) {
    const m = Math.floor((l + r + 1) / 2);
    if (m * m <= x) l = m;
    else r = m - 1;
  }
  return l;
};

Easy 70 · Climbing Stairs

Patrón: Fibonacci iterativo / DP mínima
Idea: para llegar a cada escalón podés venir del anterior o del anterior al anterior.
Pitfall: usar recursión lenta en vez de loop.
Objetivo: ≤ 120s

var climbStairs = function(n) {
  if (n <= 2) return n;
  let anterior = 1, actual = 2;
  for (let i = 3; i <= n; i++) {
    const sig = anterior + actual;
    anterior = actual;
    actual = sig;
  }
  return actual;
};

"Cada respuesta es la suma de las dos previas."

📚 Stack

Stack 20 · Valid Parentheses

Patrón: Stack
Idea: push de aperturas y validás cierres con un mapa.
Pitfall: comparar contra el tope equivocado o no validar stack vacío.
Objetivo: ≤ 120s

var isValid = function(s) {
  const pares = { ")":"(", "]":"[", "}":"{" };
  const pila = [];
  for (const c of s) {
    if ("([{".includes(c)) pila.push(c);
    else if (pila.pop() !== pares[c]) return false;
  }
  return pila.length === 0;
};

"Cuando abro, guardo. Cuando cierro, lo último guardado tiene que ser la apertura correcta."

🔗 Linked Lists

Linked List 21 · Merge Two Sorted Lists

Patrón: Dummy node + merge
Idea: dummy node evita casos especiales; siempre avanzás el menor.
Pitfall: olvidarte de devolver dummy.next.
Objetivo: ≤ 150s

var mergeTwoLists = function(l1, l2) {
  const dummy = new ListNode(0);
  let cur = dummy;
  while (l1 && l2) {
    if (l1.val < l2.val) { cur.next = l1; l1 = l1.next; }
    else { cur.next = l2; l2 = l2.next; }
    cur = cur.next;
  }
  cur.next = l1 || l2;
  return dummy.next;
};

Linked List 83 · Remove Duplicates from Sorted List

Patrón: Pointer update
Idea: si el siguiente tiene el mismo valor, lo salteás.
Pitfall: avanzar el puntero después de saltear mal y dejar duplicados.
Objetivo: ≤ 150s

var deleteDuplicates = function(head) {
  let cur = head;
  while (cur && cur.next) {
    if (cur.val === cur.next.val) cur.next = cur.next.next;
    else cur = cur.next;
  }
  return head;
};

🌳 Binary Trees

Tree 94 · Binary Tree Inorder Traversal

Patrón: DFS inorder
Idea: recursión: izquierda → raíz → derecha.
Pitfall: olvidarte del caso null o invertir el orden.
Objetivo: ≤ 150s

var inorderTraversal = function(root) {
  const res = [];
  const dfs = (node) => {
    if (!node) return;
    dfs(node.left);
    res.push(node.val);
    dfs(node.right);
  };
  dfs(root);
  return res;
};

Tree 100 · Same Tree

Patrón: Recursive tree compare
Idea: ambos null = true; uno null = false; comparás valor y ramas.
Pitfall: no separar claramente "ambos null" de "uno null".
Objetivo: ≤ 120s

var isSameTree = function(p, q) {
  if (!p && !q) return true;
  if (!p || !q) return false;
  if (p.val !== q.val) return false;
  return isSameTree(p.left, q.left)
      && isSameTree(p.right, q.right);
};

Tree 101 · Symmetric Tree

Patrón: Mirrored tree recursion
Idea: comparás espejo entre subárbol izquierdo y derecho.
Pitfall: comparar izquierda con izquierda en vez de espejo cruzado.
Objetivo: ≤ 150s

var isSymmetric = function(root) {
  const mirror = (l, r) => {
    if (!l && !r) return true;
    if (!l || !r) return false;
    return l.val === r.val
        && mirror(l.left, r.right)
        && mirror(l.right, r.left);
  };
  return mirror(root.left, root.right);
};

"Comparo espejo cruzado: izquierda.left vs derecha.right."

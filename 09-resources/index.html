📖 Recursos Curados

Los mejores libros, canales de YouTube, repos de GitHub y plataformas para dominar system design.

📚 Libros Esenciales

Ordenados por prioridad. Empieza con los primeros si eres nuevo.

📗

Designing Data-Intensive Applications

Martin Kleppmann · O'Reilly, 2017
El libro más recomendado por ingenieros senior. Cubre desde fundamentos de bases de datos hasta sistemas distribuidos avanzados: replicación, sharding, transacciones distribuidas, consistencia. No es específico de entrevistas sino de ingeniería real. Lectura obligatoria para cualquier ingeniero senior.

Avanzado
📘

System Design Interview — An Insider's Guide Vol 1

Alex Xu · ByteByteGo, 2020
El mejor libro específico para entrevistas. Framework paso a paso con 16 casos de estudio: URL shortener, rate limiter, consistent hashing, key-value store, unique ID generator, notification system, chat system, search autocomplete, YouTube, Google Drive. Muy visual y claro.

Principiante
📙

System Design Interview — Vol 2

Alex Xu & Sahn Lam · ByteByteGo, 2022
Continuación del Vol 1. Casos más complejos: nearby friends, Google Maps, distributed message queue, metrics monitoring, ad click event aggregation, hotel reservation, stock exchange. Para candidatos a Staff/Senior.

Intermedio
📕

Fundamentals of Software Architecture

Mark Richards & Neal Ford · O'Reilly, 2020
El libro definitivo sobre arquitectura de software. Cubre todos los estilos arquitecturales: microservicios, event-driven, space-based, pipeline, microkernel. Ideal para architects y senior engineers que quieren ir más allá de las entrevistas.

Intermedio
📔

Software Architecture: The Hard Parts

Neal Ford, Mark Richards, et al. · O'Reilly, 2021
Para quienes ya tienen bases sólidas. Trata los temas más difíciles: coupling vs cohesion, data ownership en microservicios, distributed transactions, cuando migrar de monolito. Usa casos narrativos para explicar las decisiones.

Avanzado
📒

Grokking the System Design Interview

Fahim ul Haq · Educative.io / DesignGurus.io
El curso más recomendado para entrevistas. Sistemático y estructurado. Cubre 22 casos de estudio. Disponible también en versión libro. Actualizado a 2026. El más popular en la comunidad.

Principiante

🗺️ Camino de Aprendizaje Recomendado

🌱 Principiante (0-3 meses)

1. Ver todos los videos de Gaurav Sen (YouTube, gratis) · 2. Leer System Design Primer (GitHub, gratis) · 3. Leer libro "System Design Interview" de Alex Xu · 4. Practicar URL Shortener y Chat System por tu cuenta

📈 Intermedio (3-6 meses)

1. Tomar Grokking the System Design Interview (DesignGurus) · 2. Ver ByteByteGo channel · 3. Practicar 2-3 casos por semana con mock interviews en Pramp · 4. Leer engineering blogs de empresas

🚀 Avanzado (6+ meses)

1. Leer "Designing Data-Intensive Applications" (Kleppmann) · 2. Seguir InterviewReady de Gaurav Sen · 3. Leer papers académicos (Dynamo, Kafka, Cassandra) · 4. Contribuir a proyectos open source de infraestructura

✅

La clave: Consistencia > Intensidad. 30 minutos diarios durante 6 meses supera a una semana de estudio intensivo. Practica en voz alta. Haz mock interviews reales. El conocimiento sin práctica no alcanza para las entrevistas.

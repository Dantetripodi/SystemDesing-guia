Quick Reference — Conceptos Clave

Los conceptos que el 80% de las entrevistas evalúan.

⚖️

CAP Theorem

Un sistema distribuido solo puede garantizar 2 de 3: Consistency, Availability, Partition Tolerance.

🔄

Consistent Hashing

Distribuir carga minimizando redistribución cuando se agregan/quitan servidores.

📦

Sharding

Particionar datos horizontalmente entre múltiples bases de datos para escalar.

🔁

Replication

Copiar datos entre nodos para alta disponibilidad. Leader/follower o multi-leader.

⚡

Caching

Almacenar respuestas frecuentes en memoria (Redis) para reducir latencia y carga en DB.

🌊

Rate Limiting

Limitar peticiones por usuario/IP para proteger el sistema. Token bucket, sliding window.

🔌

Circuit Breaker

Patrón que evita llamadas a servicios caídos, retornando error inmediato (fail fast).

📡

Message Queue

Desacoplar productores y consumidores. Kafka para streams, RabbitMQ para tasks.

Framework — Cómo responder en una entrevista

El framework de 7 pasos que usan los ingenieros de FAANG.

1

Clarificar Requisitos

Nunca asumas. Pregunta: ¿cuántos usuarios? ¿funcionalidades core? ¿latencia aceptable? Define scope.

2

Estimaciones de Escala

Calcula QPS (queries per second), storage necesario, bandwidth. Estos números guían tus decisiones.

3

Definir APIs

¿Qué endpoints expone el sistema? Define los contratos antes de hablar de implementación.

4

Modelo de Datos

¿Qué entidades existen? ¿Relaciones? ¿SQL o NoSQL? ¿Cómo se almacenan?

5

Diseño de Alto Nivel

Dibuja el diagrama general: clientes, servidores, bases de datos, caches, queues. El flujo principal.

6

Deep Dive en Componentes

El interviewer elegirá 1-2 componentes para profundizar. Ten listos los trade-offs de cada decisión.

7

Identificar Bottlenecks

¿Qué falla primero bajo carga? SPOF (single point of failure), hot spots, cuellos de botella.
